Creates an ExecutableArgument from explicitly supplied named and positional arguments, ensuring both representations are internally aligned and normalized.

This is the standard factory used by the runtime to wrap invocation contexts into a consistent format before they are passed to resolvers, generators, or hint processors.

The returned implementation guarantees:

  • Positional argument order is preserved
  • Named arguments are available in both string and symbol forms
  • No further normalization is required downstream
