Thrown when a class exists but cannot be materialized into a concrete ClassDeclaration by JetLeaf.

This exception represents a semantic failure, not a lookup failure. The class was located, but JetLeaf could not construct a stable, fully-resolved runtime representation.

Common Causes

This typically occurs when:

  • A dart:mirrors type such as ClassMirror or TypeMirror is used
  • A generic class is referenced without resolved type arguments
  • Generic type information has been erased at runtime

Why This Matters

JetLeaf requires a concrete, fully-resolved runtime type to safely perform reflection, hierarchy analysis, and code generation.

Mirror-only or erased generic types do not provide sufficient guarantees.

How to Fix

  • Use a concrete class instead of a mirror-based type
  • Annotate generic classes with @Generic(MyResolvedClass)
  • For Dart core generic types, use the fully qualified concrete type

This exception extends ClassNotFoundException to allow unified handling of class resolution failures.
