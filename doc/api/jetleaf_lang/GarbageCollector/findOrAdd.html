findOrAdd

findOrAdd abstract method

Garbage findOrAdd(
  1. String key,
  2. Object source
)

Finds an existing garbage entry for key or adds a new untyped entry if none exists.

Unlike getOrCreate, this method does not enforce generic typing and always returns a raw Garbage instance.

Example

final entry = GC.findOrAdd(
  'runtime:resolver',
  RuntimeResolver(),
);

final resolver = entry.getSource() as RuntimeResolver;

Comparison

Method Typed Overwrites Primary Use Case
getOrCreate ✅ ❌ Safe, typed lazy initialization
findOrAdd ❌ ❌ Dynamic or internal caching

Notes

  • Existing entries are never replaced
  • Cleanup policies may evict entries at any time
